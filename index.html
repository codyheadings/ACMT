AssembloidCellMigrationTracker (ACMT)

A MATLAB-based toolkit for 4D cell migration analysis in volumetric 4D assembloid live-cell imaging data.

Pipeline Overview

  1. Preprocessing - Convert raw microscopy data (.nd2) into trackable TIFF stacks.
  2. Manual Tracking (External) - Perform cell tracking in ImageJ or similar software.
  3. Data Collection - Aggregate tracking outputs into a unified dataset.
  4. Filtering (Optional) - Remove duplicate or overlapping tracks.
  5. Metric Computation - Calculate speed and displacement metrics per cell.
  6. Statistical Analysis - Compare experimental groups and compute statistics.
  7. Additional Metrics - Compute MSD and turning angles for further motion analysis.
  8. Visualization - Generate representative trajectory plots.

Pipeline Operation

Step 1 - Preprocess Raw Imaging Data

Begin with raw microscopy files (e.g., Nikon .nd2). Use preProcessND2 to extract a specific series/channel combination, apply optional spatial binning, and perform Gaussian background subtraction. Save to a shared tracking folder.

The output is a cleaned BigTIFF file suitable for manual cell tracking. 

preProcessND2("data/example.nd2", "output", ...
    SeriesIndex=1, ChannelIndex=2);

Step 2 - Perform Manual Cell Tracking (External)

Load the processed TIFF into ImageJ (or equivalent image software) and perform cell tracking using OrthoTrack, saving cell trajectories as ROIs. Export saved results as Results.csv.

Each tracker's data should be stored in their own folder. Example: 

rootFolder/
├── Channel1/
│   ├── Dataset1/
│   │   ├── Series1/
│   │   │   ├── Tracker1/
│   │   │   │   └── Results.csv
│   │   │   ├── Tracker2/
│   │   │   │   └── Results.csv

Step 3 - Collect Tracking Data

When all cells have been tracked across all images, use collectTrackingData to combine all tracking outputs into a single table. This step and the following ones can be performed on our tracking coordinate data, located here.

Folder names are automatically converted into grouping variables (Group1, Group2, etc.). 

tracks = collectTrackingData("example", ...
    XYScale=0.65, ZScale=20, TScale=30);

Step 4 - (Optional) Remove Duplicate Tracks

If multiple trackers labeled the same cells, you can use filterDuplicateTracks to remove redundant trajectories based on detected spatial overlap. This is not essential and there are some cases where you may want to leave the data unfiltered. 

tracks = filterDuplicateTracks(tracks);

Step 5 - Compute Cell Metrics

Run computeTrackingMetrics to calculate per-cell movement statistics, including:

  • Speed (average, cumulative, variance)
  • Distance (total, net displacement)
  • Directionality ratio
results = computeTrackingMetrics(tracks, "output");

Output files are generated per top-level group (e.g., per channel).

Step 6 - Statistical Analysis

Although not used in our process, analyzeByGroup can be used to compare experimental groups and compute statistical significance (e.g., Shapiro-Wilk normality tests and pairwise comparisons). 

analysis = analyzeByGroup(results.C1, ...
    "/swtest", "Dataset", "output/analysis.xlsx");

Step 7 - Additional Motion Analysis

Mean Squared Displacement (MSD)

Use computeMSD to quantify diffusive behavior of cells.

Important: This function requires the @msdanalyzer MATLAB toolkit (developed by Jean-Yves Tinevez). Ensure it is installed and added to your MATLAB path. 

mmsd = computeMSD("tracks.xlsx", "output");
Turning Angles

Use computeTurningAngles to measure directional changes between consecutive movement steps. 

angles = computeTurningAngles("tracks.xlsx", "output");

Step 8 - Visualization

Generate representative trajectory plots using plotRepresentativeGraphs. Tracks are selected based on median path length for visualization consistency. 

plotRepresentativeGraphs("example/AggregatedResults.xlsx", ...
    "figures", 8, 0);

Script Documentation

Detailed descriptions of each script. Click on one to expand its content.

Description:
preProcessND2 loads data from a .nd2 Nikon Microscope Image file using Bio-Formats, applies optional spatial binning, performs Gaussian background subtraction, and saves the processed file as a BigTIFF.

Input:

  • inputFile - path to a .nd2 file
  • outputFolder - directory where output TIFF will be saved
  • options (optional):
    • SeriesIndex - .nd2 series to process
    • ChannelIndex - channel to extract
    • BinSize - spatial binning factor
    • GaussianSigma - smoothing kernel for background subtraction
    • Logs - enable console output

Output:

  • Saves a processed BigTIFF file to outputFolder

Processing Steps:

  • Load .nd2 file via Bio-Formats
  • Select specified series and channel
  • Apply spatial binning (optional)
  • Apply 3D Gaussian background subtraction
  • Reformat data to (X, Y, Z, C, T)
  • Export as BigTIFF

Notes:

  • Requires Bio-Formats (bfGetReader, bfsave)
  • Uses parallel processing (parfor) for background subtraction
  • Large datasets may require significant memory

Usage Example:

preProcessND2( ...
    "data/sample.nd2", ...
    "output", ...
    'SeriesIndex', 1, ... % or SeriesIndex = 1
    'ChannelIndex', 2, ...
    'BinSize', 2 ...
    ...
);

% Output: Processed_S1_C2.tif saved to "output"

Description:
compileResultsCSV gathers tracking results from multiple subdirectories into a single table for use in the pipeline. Each subdirectory is expected to contain a results file (default: Results.csv) exported from ImageJ or similar tools. Optional (but recommended) X, Y, Z, and Time unit conversions can be applied during compilation.

Input:

  • inputFolder - a directory containing tracker subfolders
  • outputFolder (optional) - a directory where compiled results will be saved
  • options (optional):
    • XYScale - a scaling factor for X and Y. Example: 0.65 would convert 1 ImageJ pixel to 0.65 micrometers.
    • ZScale - a scaling factor for Z. Example: 20 would convert 1 ImageJ slice to 20 micrometers.
    • TScale - a scaling factor for time. Example: 30 would convert 1 ImageJ frame to 30 minutes.
    • OutputFilename - the name of output file
    • ResultsFilename - the expected filename in each subfolder. Default = "Results.csv"

Output:

  • compiledData - a table containing all tracking points across all trackers. Columns:
    • Num - the row index within original file
    • TrackerName - the source subfolder name
    • X, Y, Z - spatial coordinates (converted)
    • T - time (converted)

Expected Folder Structure:

inputFolder/
├── Tracker1/
│   └── Results.csv
├── Tracker2/
│   └── Results.csv

Usage Example:

compiledData = compileResultsCSV( ...
    "data/inputFolder", ...
    "data/outputFolder", ...
    'XYScale', 0.65, ... % or XYScale = 0.65
    'ZScale', 20, ...
    'TScale', 30 ...
    ...);

head(compiledData) % Preview results

Description:
cellDistances computes displacement-based metrics for each tracked cell, including total distance traveled, average frame-to-frame displacement, net displacement, and the directionality ratio. Operates on compiled tracking data (from compileResultsCSV.m) and uses segmentTracks to separate individual trajectories.

Input:

  • compiledData - a table containing at minimum: X, Y, Z, T
  • outputFolder (optional) - directory to save results
  • options (optional):
    • OutputFilename - the output file name
    • Logs - enable/disable console output

Output:

  • distanceTable - a table with one row per cell track. Columns:
    • Cell - track index
    • AvgFrameDistance - mean displacement between frames
    • CumulativeDistance - total distance traveled across all frames
    • RelativeDisplacement - net displacement between first and last frame
    • DirectionalityRatio - Net displacement / total path length

Notes:

  • Uses segmentTracks to separate trajectories
  • Tracks must be time-ordered
  • Requires consistent units across X/Y/Z

Usage Example:

data = readtable('compiledResults.xlsx');

distanceTable = cellDistances(data, "output");

head(distanceTable) % preview results

Description:
cellSpeeds calculates speed-based metrics for each tracked cell using compiled tracking data. It computes average speed, cumulative speed, and variance across the full trajectory as well as split into first and second halves.

Input:

  • compiledData - table with columns: X, Y, Z, T
  • outputFolder (optional) - directory for saved results
  • options (optional):
    • OutputFilename - output file name
    • Logs - enable/disable logging

Output:

  • speedTable - table with one row per cell track. Columns:
    • Cell - track index
    • AvgCellSpeed - mean speed
    • CumulativeCellSpeed - total speed
    • VarianceEntirePeriod - variance across full track
    • VarianceFirstHalf - variance (first half)
    • VarianceLastHalf - variance (second half)

Notes:

  • Assumes constant frame interval across all tracks
  • Uses Euclidean distance between consecutive points
  • Uses segmentTracks to separate trajectories

Usage Example:

data = readtable('compiledResults.xlsx');

speedTable = cellSpeeds( ...
    data, ...
    "output", ...
    struct('Logs', true) ...
);

mean(speedTable.AvgCellSpeed) % average across all cells

Description:
collectTrackingData traverses a directory tree of raw tracking results and compiles all coordinates into a single grouped table, one row per timepoint.

Each folder level above the tracker directories is treated as a grouping category and automatically mapped to a column (Group1, Group2, etc.) in the output. Unit scaling is applied during this step by the (optional) given conversion ratios.

Example Folder Structure:

rootFolder/
├── Channel1/
│   ├── Dataset1/
│   │   ├── Series1/
│   │   │   ├── Tracker1/
│   │   │   │   └── Results.csv
│   │   │   └── Tracker2/
│   │   │       └── Results.csv

Inputs:

  • rootFolder - root of the folder tree
  • options (optional) -
    • XYScale, ZScale, TScale - unit scaling factors
    • ResultsFilename - expected tracker file name (default: Results.csv)
    • OutputFile - if provided, saves the compiled table to this path
    • Logs - enable console output

Output:

  • aggregatedTracks - table with one row per timepoint across all trackers. Columns:
    • Group Columns (Group1...GroupN)
    • Num, TrackerName
    • X, Y, Z, T in scaled units

Usage Example:

% Collect all tracks with unit conversion applied
tracks = collectTrackingData('/data/root', ...
    XYScale=0.65, ...
    ZScale=20, ...
    TScale=30);

% Save the raw track table to '/output/'
tracks = collectTrackingData('/data/root', ...
    XYScale=0.65, ZScale=20, TScale=30, ...
    OutputFile='/output/AggregatedTracks.xlsx');

Notes:

  • The directory tree must be at least two levels deep (one group level plus a tracker directory)
  • Tree depth is inferred from the first Results.csv found. All tracker directories must be at the same depth
  • Group labels are inferred automatically from folder names and can be renamed manually in the output table
  • The output table is the expected input for filterDuplicateTracks and computeTrackingMetrics

Description:
filterDuplicateTracks identifies and removes duplicate or highly similar cell tracks from compiled tracking data, in the case that multiple trackers have annotated the same cell.

The function compares tracks based on spatial similarity and removes duplicates according to optional threshold arguments. The cleaned dataset is returned as a table and can optionally be saved with a summary.

Inputs:

  • aggregatedTracks - a table containing tracking data from (must include X, Y, Z, T) (or a path to a file that contains the table).
  • options (optional)
    • XYTolerance - Maximum mean XY distance between two tracks at shared timepoints for them to be considered duplicates.
    • ZTolerance - Maximum mean Z distance between two tracks at shared timepoints for them to be considered duplicates.
    • OverlapPercent - Minimum fraction of shared timepoints at which both the XY and Z tolerances must be satisfied for tracks to be flagged as duplicates. Must be in (0, 1].
    • OutputPath - If provided, a two-sheet Excel report is written to this path:
      • KeptTracks - summary of all tracks that were retained
      • RemovedTracks - summary of removed tracks with the index of the track they were considered a duplicate of
    • Logs - enable console output

Output:

  • filteredData - aggregatedTracks with duplicates removed

Usage Example: 

data = readtable("compiledResults.xlsx"); 
% Load compiled tracking data

filtered = filterDuplicateTracks(data); 
% Remove duplicate trajectories

Notes:
This function is typically used after compileResultsCSV and before downstream analysis (e.g., MSD or turning angles).

Description:
computeTrackingMetrics takes a compiled coordinate table from collectTrackingData (or filterDuplicateTracks) and computes per-cell distance and speed metrics for every tracker. Results are grouped by the first group column and written to one .xlsx file per group.

For each detected grouping (e.g., Channel/Dataset/Series), this function:

  • Computes per-cell distance metrics using cellDistances
  • Computes per-cell speed metrics using cellSpeeds
  • Merges results into one table per top-level group

Group columns are detected positionally: any column preceding Num in the input table is treated as a grouping column, regardless of name. The first group column determines the output file split.

Inputs:

  • aggregatedTracks - coordinate table from collectTrackingData
  • outputFolder - directory to save output files
  • options (optional) -
    • OutputFilename - name appended to the Group1 value to form the filename (default: _CombinedResults.xlsx)
    • Logs - enable console output

Output:

  • aggregatedData - struct with one field per top-level group
    • Group Columns (Group1...GroupN)
    • TrackerName, Cell
    • Speed metrics: AvgCellSpeed, CumulativeCellSpeed, VarianceEntirePeriod, VarianceFirstHalf, VarianceLastHalf
    • Distance metrics: AvgFrameDistance, CumulativeDistance, RelativeDisplacement, DirectionalityRatio

Usage Example:

% Pass tracks directly from collectTrackingData
tracks  = collectTrackingData('/data/root', XYScale=0.65, ZScale=20, TScale=30);
results = computeTrackingMetrics(tracks, '/output');

% With optional duplicate filtering step in between
tracks  = collectTrackingData('/data/root', XYScale=0.65, ZScale=20, TScale=30);
tracks  = filterDuplicateTracks(tracks);
results = computeTrackingMetrics(tracks, '/output');

% Access one group's results
c1Table = results.C1;

Notes:

  • If filterDuplicateTracks is used between collection and metric computation, the filtered table is passed directly here with no other changes needed
  • Output files are named <Group1Value>_CombinedResults.xlsx by default

Description:
analyzeByGroup performs grouped statistical analysis on aggregated tracking results. It organizes data by group and computes summary metrics for each group.

Inputs:

  • aggregatedResults - table field produced by computeTrackingMetrics
  • swTestPath - path to a folder containing swtest.m (Shapiro-Wilk test). Will throw an error if invalid.
  • compareGroup - the name of the column group that will be the comparison group. Example: Using Channel, your data would be compared across channels. (As long as there is indeed a 'Channel' column in the input table)
  • outputFile - full file path for saving results
  • options (optional) -
    • Metrics - variables to summarize (e.g., AvgCellSpeed, CumulativeDistance). Defaults to all available metrics.
    • GroupColumns - Names of columns to use as descriptors in the wide-format metric sheets (e.g. {'Group1','Group2','Group3','Group4'}). Auto-detected if not supplied.
    • Logs - enable console output (default: true)

Output:

  • analysisResults - struct with fields:
    • NormalityTests - table with columns Metric, Group, h_value, p_value. One row per metric-group combination from the selected category.
    • MetricTables - struct with one field per metric, each field being a table of raw data grouped by Group columns with an average for that group at the end. Useful for making graphs from the raw data in other programs.
    • PairwiseTests - struct with one field per unique group pair. Each field contains a comparison table for the chosen groups with a row for each metric comapred, alongside columns for p and h values, and group means and medians.

Usage Example: 

% aggregatedTracks is a table with all tracking data
results = computeTrackingMetrics(aggregatedTracks, '/output')
c1Analysis = analyzeByGroup(results.C1, '/swtest', 
    ... 'Dataset', '/output/C1AnalysisResults.xlsx') % Compare each dataset in Channel 1
c2Analysis = analyzeByGroup(results.C2, '/swtest', 
    ... 'Series', '/output/C1AnalysisResults.xlsx') % Compare each series in Channel 2

% Inspect a specific pairwise sheet
c1Analysis.PairwiseTests.Dataset11_vs_Dataset12

Description:
computeMSD calculates the Mean Squared Displacement (MSD) from 3D cell tracking data. It supports both table input and file-based input, automatically segmenting trajectories using segmentTracks and computing ensemble MSD statistics.


This function relies on the external @msdanalyzer MATLAB toolkit (developed by Jean-Yves Tinevez).
** You must have this package installed and added to your MATLAB path before you can use this function. **

The function optionally generates a mean MSD plot with error bars and can save the figure to disk.


Further steps:

With this msdanalyzer object, you can fit lines to calculate the diffusion coefficient or use velocity autocorrelation to make informative graphs. Learn more at @msdanalyzer.

Inputs:

  • inputData - table or file path containing columns: X, Y, Z, T
  • outputFolder - (optional) directory to save output figure
  • options (optional) -
    • OutputFilename - name of saved figure
    • CreateFigure - toggle plotting
    • Logs - enable console output

Output:

  • mmsd - Nx4 array from @msdanalyzer:
    • Column 1: time lag
    • Column 2: mean MSD
    • Column 3: standard deviation
    • Column 4: number of samples

Usage Example: 

% Load tracking data from file
mmsd = computeMSD('/example/data/tracks.xlsx');

% Use an existing table and save a figure
data = readtable('tracks.xlsx');
mmsd = computeMSD(data, '/output/folder', ...
    OutputFilename="MyMSD.fig");

Description:
computeTurningAngles calculates turning angles between consecutive displacement vectors for each tracked cell in 3D space. While not directly used in our cell tracking pipeline, the function may provide some use for auxillary tracking metrics.

The function first segments trajectories using segmentTracks, then computes the change in trajectory angle between successive motion vectors using the dot product. Angles are reported in degrees.

Any steps with zero-length displacement vectors are skipped to avoid division-by-zero errors.

Inputs:

  • inputData - table or file path containing T, X, Y, Z
  • outputFolder - (optional) directory to save results
  • options (optional) -
    • OutputFilename - output CSV file name
    • Logs - enable console output

Output:

  • angleTable - table with:
    • CellNum - track index
    • TurningAngle_deg - angle between steps

Usage Example: 

% Compute angles from a file
angles = computeTurningAngles('/example/data/tracks.xlsx');

% Compute and save results
data = readtable('tracks.xlsx');
angles = computeTurningAngles(data, '/output', ...
    OutputFilename="Angles.csv");

Description:
plotRepresentativeGraphs visualizes representative cell tracks in 3D. Tracks are sorted by total path length, and a subset near the median is selected for plotting. Two figures are generated: (1) cumulative displacement over time, and (2) net displacement vectors.

Input:

  • inputFile - path to aggregated tracking results file
  • outputFolder (optional) - directory to save figures
  • numTracks - number of tracks to plot
  • offset - index offset from median track selection
  • options (optional):
    • FileType - output file format (e.g., .fig, .png)
    • ShowAxes - toggle axis labels
    • ShowTitle - toggle titles
    • TimeScale - scaling factor for time values
    • SpaceUnit - units for spatial axes
    • TimeUnit - units for time colorbar
    • Logs - enable console output

Output:

  • fig1 - cumulative displacement figure
  • fig2 - net displacement vector figure

Notes:

  • Uses segmentTracks internally
  • Tracks are centered to start at the origin
  • Time is visualized using a color gradient
  • Selection is based on median track length

Usage Example:

[fig1, fig2] = plotRepresentativeGraphs( ...
    "AggregatedResults.xlsx", ...
    "figures", ...
    8, ...
    0, ...
    'TimeScale', 60, ...
    'TimeUnit', "hr" ...
    ...
);

% fig1 = cumulative displacement
% fig2 = net displacement vectors

Description:
segmentTracks splits a table of time-ordered tracking coordinates into individual cell trajectories. A new track is assumed to begin whenever the time column (T) resets to 0.

Input:

  • data - a MATLAB table containing at minimum the columns: X, Y, Z, T

Output:

  • tracks - a cell array of tables, where each cell contains one trajectory

Notes:

  • Input data must be sorted in time order
  • Each track must start with T = 0
  • Tracks shorter than 1 frame are ignored

Usage Example:

data = readtable('example/directory/trackingData.xlsx');

tracks = segmentTracks(data);

% Access first track
track1 = tracks{1};
plot3(track1.X, track1.Y, track1.Z);
