# Tracking Data and Coordinates
Example data provided by the Zeldich lab gathered from 24 forebrain assembloids using the OrthoTrack manual tracking pipeline. This data can be used as an example for tracking results formatting, as well as inputs to any of the pipeline functions. There are two parts to this data: 

1) an example folder containing channel, dataset, and series subfolders, each with tracking data from 3+ trackers in each path. These are raw ImageJ output CSV files (Results.csv) organized by dataset, series, and channel in the repository folder structure, which serve both as the direct output of the OrthoTrack tracking process and as a layout reference for users applying the pipeline to new data. They can be processed using the provided compile scripts to generate analysis-ready tables.

2) an AggregatedTracks.xslx file is given, with all the data from the Results.csv files converted and grouped into one table. The cell tracks are consolidated across datasets with coordinates converted to physical units (µm, minutes), directly suitable for MSD fitting and migration metric computation. Users wishing to use the coordinates for image-based cell detection model training should note that pixel-unit coordinates can be recovered by applying the inverse of the calibration factors documented in the repository (0.65 µm/pixel laterally, 20 µm/slice axially).
